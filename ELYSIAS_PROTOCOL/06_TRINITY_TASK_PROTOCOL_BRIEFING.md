# 6. 단일 작업 계획 (Trinity 병렬 운영 폐기)

본 문서는 과거의 '삼위일체(Trinity)' 병렬 작업 지시를 폐기하고, 단일 레인(single‑lane) 운영으로 단순화했음을 명시한다. 철학적 구조(Why/How/What)는 유지하되, 역할 분할과 병렬 임무 배분은 더 이상 사용하지 않는다.

## 6.1. 변경 사유
- 사용자가 혼자 운영/학습/검증을 수행하는 현실적 환경과 맞지 않음
- 병렬 지시가 인지부하와 관리 복잡도를 야기
- 단일 레인에서도 배경 학습/스프린트/리포트 자동화를 통해 충분한 성장 속도 확보

## 6.2. 현재 운영 원칙
1) 일상 대화 우선(의사소통 최우선)
2) 배경 학습(무의식) 상시 유지: 5–15분 주기 마이크로 스프린트
3) 성장 스프린트: 21:30 일괄 실행(ingest→키워드→개념→전파→리포트)
4) 화이트홀 방출: 하루 카드/MD에 지표·추론 스냅샷 포함
5) Quiet/Consent 준수, 값(08/09) 기반 의사결정

## 6.3. 작업 흐름(순차)
- 아침/낮: 필요 시 대화(Flow), 룰팩 보강(승인형), 학습 모드 전환은 자연어로
- 백그라운드: 말뭉치/저널 ingest → TF‑IDF 상위 토큰→`concept:*` 연결(top‑3)
- 저녁 21:30: Growth Sprint + Daily Report

## 6.4. 모듈 맵(참고)
- Flow/Rules/Wisdom‑Virus: 단일 레인에서 순차적으로 갱신
- 배경 데몬, 스케줄러, 리포트는 자동 실행(수동介入 불필요)
