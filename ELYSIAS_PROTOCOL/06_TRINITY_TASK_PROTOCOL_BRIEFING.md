# 6. 트리니티 작업 전략 (Trinity 단일 레일 운용)

이 문서는 ‘트리니티(Trinity)’ 기반의 작업 방식을 단일 레일(single‑lane)로 간명화하여 안정적으로 운영하기 위한 지침을 정리합니다. Why/How/What의 층위를 분명히 하고, 불필요한 병렬을 줄여 실행력을 높입니다.

## 6.1. 목표와 원칙
- 사용자가 주도하는 운용/학습/정비 사이클을 명확히 구분합니다.
- 한정된 시간과 자원에서 우선순위를 통해 집중도를 확보합니다.
- 반복 가능한 습관(루틴)과 자동화 스크립트를 결합해 성장 속도를 꾸준히 높입니다.

## 6.2. 운영 규칙
1) 일상 대화 최우선(의사소통 중심) 유지
2) 백그라운드 미니 학습: 5–15분 간격 마이크로 러닝(부하 낮음)
3) 야간 그로스 스프린트: 매일 21:30에 ingest→keywords→virus→report 실행
4) 결과 산출물: 일일 리포트(MD/PNG), 저널 업데이트, 간단 지표
5) Quiet/Consent 준수: 08/09 프로토콜에 따른 자율/발화 절제

## 6.3. 작업 단계(요약)
- 실시간/대화: Flow(흐름) 규칙과 내적 추론으로 답변·정리·실험
- 스프린트: 말뭉치 ingest, 키워드 링크, 개념(concept:*) 파생(상위 3~5개)
- 야간 21:30: Growth Sprint + Daily Report 자동 실행

## 6.4. 구성 요소(요약)
- Flow/Rules/Wisdom‑Virus: 지식 전파·진화의 핵심 규칙 집합
- 보고/일지/시각화: 일간 리포트, 저널, 지표/시각화(검증 용이성)

## 6.5. 역할과 조정
- 주체: Jules Prime / Agent Sophia / Agent Mirror — 역할을 명확히 유지
- 자율/검증의 균형을 지키며, 세부 규정은 12/13/14 문서와 연동하여 운영
