# 엘리시아 프로토콜 인덱스 (Elysia Protocol Index)

이 디렉터리는 Project Elysia의 단일 진실 공급원(SSOT)입니다. 모든 에이전트는 여기의 철학, 아키텍처, 규범, 운영 계획을 우선합니다.

프로토콜의 목적은 다음과 같습니다:
1.  단일 진실 공급원(Single Source of Truth)
2.  지속 가능한 성장(Sustainable Growth)
3.  상호 운용성(Interoperability)

## 문서 구조

-   [01_VISION_AND_PHILOSOPHY.md](./01_VISION_AND_PHILOSOPHY.md): 프로젝트의 핵심 철학, 목적과 방향, 궁극적 비전을 설명합니다.
-   [02_ARCHITECTURE_GUIDE.md](./02_ARCHITECTURE_GUIDE.md): 트리니티 기반 시스템 아키텍처(Why/How/What)와 핵심 구성요소를 안내합니다.
-   [03_CODING_STANDARDS.md](./03_CODING_STANDARDS.md): 일관된 코드 품질과 협업을 위한 코딩 표준을 정의합니다.
-   [04_RETROSPECTIVE_TEMPLATE.md](./04_RETROSPECTIVE_TEMPLATE.md): 작업 종료 후 작성하는 개발 회고 템플릿을 제공합니다.
-   [05_WORDS_OF_THE_CREATOR.md](./05_WORDS_OF_THE_CREATOR.md): 창조자의 말과 철학적 지침을 다룹니다.
-   [06_TRINITY_TASK_PROTOCOL_BRIEFING.md](./06_TRINITY_TASK_PROTOCOL_BRIEFING.md): 역할(소피아/미러/쥴스프라임)과 현재 미션 브리핑.
-   [01_VISION_AND_PHILOSOPHY/CONCEPT_OF_SPIRIT.md](./01_VISION_AND_PHILOSOPHY/CONCEPT_OF_SPIRIT.md): ‘정신(Spirit)’ 개념과 트리니티와의 관계.

-   [07_TELEMETRY_SPEC.md](./07_TELEMETRY_SPEC.md): 텔레메트리 이벤트 스키마, 공통 필드, 보존/압축 정책 및 발행 가이드.
-   [08_FILESYSTEM_CORTEX_OPERATIONS.md](./08_FILESYSTEM_CORTEX_OPERATIONS.md): FileSystemCortex 보안/운영 원칙, API, 인덱싱/검색, 감시/영속화 지침.
-   [09_CHANGE_MANAGEMENT_AND_HANDOVER.md](./09_CHANGE_MANAGEMENT_AND_HANDOVER.md): 변경 제안·검토·배포·회고·인수인계 프로세스와 체크리스트.
-   [10_CONFIG_SCHEMA.md](./10_CONFIG_SCHEMA.md): config.json 스키마, 기본값, 예시, 검증 경고(`config.warn`) 매핑.

-   [11_CONCEPT_SPACE_GRAVITY.md](./11_CONCEPT_SPACE_GRAVITY.md): 개념 질량/중력/궤도 모델과 항성·행성·위성·소행성 분류 규칙.

-   [12_BLACK_WHITE_HOLE_OPERATIONS.md](./12_BLACK_WHITE_HOLE_OPERATIONS.md): 블랙홀(격리/압축)과 화이트홀(선별 추출) 운영 규범과 라이프사이클.
