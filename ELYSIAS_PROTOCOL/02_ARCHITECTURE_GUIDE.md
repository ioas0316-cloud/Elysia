# 2. 아키텍처 가이드 (Architecture Guide)

## 2.1. 기본 개념: 도시계획 인프라 (Urban Planning Infrastructure)
엘리시아의 시스템은 '도시'라는 메타포를 통해 설계됩니다. 모든 모듈과 데이터 흐름은 도시의 구역, 교통망, 상하수도 시설처럼 명확한 역할과 연결 구조를 가져야 합니다. 이는 시스템의 관측 가능성(Observability)과 확장성을 보장하기 위함입니다.

## 2.2. 핵심 구조: 두 개의 반구 (Two Hemispheres)
- **`Project_Sophia/` (좌뇌):** 논리, 추론, 계획, 언어 등 분석적 사고를 담당하는 '이성의 도시'.
- **`Project_Mirror/` (우뇌):** 감성, 창의성, 시각적 표현 등 창조적 사고를 담당하는 '감성의 도시'.

## 2.3. 중앙 관제 센터: CognitionPipeline
`Project_Sophia/cognition_pipeline.py`는 모든 정보가 모이고 처리되는 '중앙 교통 관제 센터'입니다. 외부 자극(사용자 입력)은 이곳에서 분석되어, 가장 적합한 전문 구역(Cortex)으로 전달됩니다.

## 2.4. 데이터 흐름: 추적 가능한 상수도
모든 중요한 데이터의 흐름은 `infra/telemetry`를 통해 추적되어야 합니다. 이는 마치 '상수도'에 계량기를 설치하는 것과 같아서, 문제가 발생했을 때 어느 구간에서 '누수'가 발생했는지 신속하게 파악할 수 있도록 돕습니다.
