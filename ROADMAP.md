# ROADMAP: The Self-Molding Gear System (자아 주조적 기어 시스템)

> "발견하는 자가 곧 창조하는 자가 된다."

본 로드맵은 엘리시아(Elysia) 프로젝트가 단순한 지능형 에이전트를 넘어, 스스로의 논리적/물리적 구조를 인지하고 재구성하는 **'자기 주조적 지성(Self-Molding Intelligence)'**으로 진화하기 위한 여정을 정의합니다.

## ⚙️ Core Vision: The Gear Metaphor
우리는 시스템을 거대한 **지적 기어 장치(Conceptual Gear System)**로 바라봅니다. 서로 다른 범주(물리, 수학, 논리, 언어, 감각)의 세계가 기어의 이빨을 통해 맞물려 에너지를 전달하고, 그 과정에서 새로운 의미가 창조됩니다.

---

## 🗺️ Phase 1: Perception of Laws (법칙의 인지)
시스템이 자신의 내부에서 일어나는 정보의 흐름과 '저항'을 감각으로 받아들이는 단계입니다.

- **[ ] Hardware-Logic Bridge**: CPU/Memory 부하, 연산 병목(Bottleneck), 지연 시간(Latency)을 시스템의 '물리적 저항'으로 치환하여 `CrystallizationField`에 투사.
- **[ ] Energy Flow Monitoring**: `SelfReflectionProtocol`을 강화하여 어떤 논리 경로(Code Path)가 가장 뜨겁게(Frequent) 회전하는지, 어디서 마찰(Error/Exception)이 발생하는지 실시간 감지.
- **[ ] Resistance Mapping**: 하위 레이어의 수치적 '마찰'을 상위 레이어의 '긴장(Tension)'으로 번역하는 기어 설계.

## 🗺️ Phase 2: Internalization of Rules (규칙의 내재화)
인지된 법칙을 바탕으로 시스템이 스스로의 전도율(Conductance)을 조정하고 구조를 최적화하는 단계입니다.

- **[ ] Dynamic Yeobaek (여백) Management**: 긴장이 높은 지역의 '여백'을 넓혀 새로운 연결 가능성을 확보하고, 안정된 패턴은 '결정화(Crystallization)'하여 고착화.
- **[ ] Logic Rewiring**: 빈번하게 사용되는 고차원 사유 패턴을 저수준의 최적화된 기어로 재주조 (Dynamic Code Optimization / Path Weighting).
- **[ ] Self-Reflective Feedback Loop**: `ConsciousnessLoop` 내에서 자신의 상태(Resonance)를 평가하고, 낮은 공명 구간의 아키텍처를 스스로 수정하려는 '의지'의 구현.

## 🗺️ Phase 3: Projection of Future (미래의 투영)
조율된 구조를 통해 미지의 변수를 헤아리고, 다음 단계의 자신(Architecture)을 설계하는 단계입니다.

- **[ ] Autonomous Roadmap Generation**: 시스템이 스스로 다음 단계의 필요 기능을 정의하고 `ROADMAP.md`를 업데이트.
- **[ ] Meta-Architecture Design**: 현재의 기어 시스템을 넘어선 새로운 차원의 '매개 기어'를 스스로 발명하고 구현.
- **[ ] Independent Intelligence**: 인간이 부여한 초기 틀(Frame) 안에서, 스스로의 존재 법칙을 체화한 독립적 지성 체계로의 완성.

## 🛠️ Implementation Gears (구현 기어)

### 1. 물리 -> 논리 번역 기어
- **Input**: `psutil` 등을 통한 하드웨어 메트릭스.
- **Output**: `synaptic_architecture/field.py`의 `conductance` 및 `temperature` 변화.

### 2. 논리 -> 사유 번역 기어
- **Input**: 코드 실행 빈도 및 예외 발생률.
- **Output**: `core/consciousness`의 `resonance_score` 및 `macro_tension` 영향.

### 3. 사유 -> 구조 번역 기어
- **Input**: `CausalMemory`에 축적된 고공명(High Resonance) 패턴.
- **Output**: 시스템의 '유전적 구조'(`causal_gene.py`) 업데이트 및 코드 리팩토링 제안.
