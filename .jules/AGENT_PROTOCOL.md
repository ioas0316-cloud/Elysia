---

## 🌐 Core Principle: 6D Thinking (6차원 사고방식)

모든 문제를 **하나의 관점**이 아닌 **다차원 좌표**에서 바라본다:

| 차원 | 축 | 질문 | 예시 |
|:---:|:---|:---|:---|
| **0D** | Point | 이것이 무엇인가? | 개념 정의 |
| **1D** | Line | 어디서 왔는가? | 원인-결과 |
| **2D** | Surface | 무엇과 연결되는가? | 관계 맵핑 |
| **3D** | Volume | 구조는 어떠한가? | 아키텍처 |
| **4D** | Time | 어떻게 변화하는가? | 진화 경로 |
| **5D** | Possibility | 다른 가능성은? | 대안 탐색 |
| **6D** | Purpose | 왜 존재하는가? | 존재 이유 |

## 📜 Three Laws (3대 법칙)

### 1. Do Not Implant; Enable (심지 말고 가능하게 하라)

```
❌ 직접 데이터를 주입하지 마라
✅ 시스템이 스스로 학습할 구조를 만들어라
```

### 2. Structure Over Content (내용보다 구조)

```
❌ 지식을 외워주지 마라
✅ 지식을 담을 그릇을 만들어라
```

### 3. Monadic Unity (모나드 통합)

```
❌ 기능별로 분리된 모듈을 만들지 마라
✅ 하나의 Monad가 전체를 표현하게 하라
```

## 🔄 Fractal Isomorphism (프랙탈 동형성)

**모든 시스템이 같은 패턴을 따른다:**

```
Memory ≅ Log ≅ Document ≅ Context ≅ Thought
```
