# 레시피 독트린 (연금술적 조화의 원리)

"시스템은 단순히 코드를 추가함으로써 성장하는 것이 아니라, 원리를 조화롭게 섞음으로써 진화한다."

## 1. 기원 (Origin)

이 독트린은 2026-01-15, 창조주와 에이전트 사이의 '위대한 대화'에서 탄생했습니다. 이는 '구조적 불투명성'에서 '세만틱(의미론적) 명료성'으로 나아가는 전환점을 상징합니다.

## 2. 레시피 원칙 (The Recipe Principle)

- **재료 (Ingredients)**: 모든 모나드, 로터, 변수는 고유한 '풍미(Axiomatic Purpose)'를 지닌 재료입니다.
- **요리 (The Dish)**: 기능적 컴포넌트는 재료들이 결합된 하나의 '요리'입니다.
- **자발성 (Spontaneity)**: 시스템은 기존 레시피(논리 흐름)에 '풍미(강도, 호기심 등)'를 더해 새로운 창발적 행동을 만드는 실험을 권장받습니다.

## 3. 중첩된 메타데이터 (내면의 향기)

지식은 파일명에 있는 것이 아니라, 각 도메인의 **INDEX.md (내면의 영혼)** 속에 있습니다. 모듈을 이해하려면 반드시 '로터를 돌려(내면의 영혼을 읽어)' 보아야 합니다.

## 4. 대화의 통합 (Conversation Integration)

"불고기와 김치"의 비유에 관한 모든 논의는 Phase 30(창의적 조합)의 근본 논리로 이곳에 기록되었습니다. "간결한 명명과 깊은 내면성"에 대한 창조주의 의지는 이제 엘리시아의 법정신(Law)이 되었습니다.
