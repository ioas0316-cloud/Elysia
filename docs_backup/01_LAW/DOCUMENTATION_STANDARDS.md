# 에토스: 문서화 표준 (Ethos: Documentation Standards)

"명확한 기록은 존재의 가장 강력한 증거이다."

이 문서는 엘리시아의 '900페이지 발전소 가이드'가 중구난방이 되지 않도록 규정하는 핵심 구조 설계도입니다.

## 1. 7단계 프랙탈 위계 시스템 (The 7-Level Fractal Hierarchy)

엘리시아의 지식은 $7 \times 7$ 의 자발적 증식 구조를 지향하며, 총 7단계의 깊이로 세분화됩니다. 이 구조는 단순한 코드 관리를 넘어 우주의 원리를 압축하여 저장하기 위한 **'거대 지식 발전소'**의 설계도입니다.

| 단계 | 명칭 (Level Name) | 주소 예시 (Example) | 포함 내용 |
| :--- | :--- | :--- | :--- |
| **L0** | **도메인 (Domain)** | `D.03_WORLD` | 8대 근원 영역 (현실, 정신, 율법 등) |
| **L1** | **권역 (Province)** | `D.03.10_PHYSICS` | 대분류 (물리, 화학, 예술, 파동 등) |
| **L2** | **군집 (Cluster)** | `D.03.11_WAVES` | 중분류 (광학, 유체 역학, 색채학 등) |
| **L3** | **기관 (Organ)** | `L3.LIGHT_ENGINE` | 주요 시스템 및 엔진 (가시화 엔진 등) |
| **L4** | **분자 (Molecule)** | `L4.REFRACTION` | 주요 알고리즘 및 논리 블록 |
| **L5** | **원소 (Ingredient)** | `VAR.REF_INDEX` | 변수, 상수, 데이터 타입 (굴절률 등) |
| **L6** | **양자 (Quantum)** | `AXIOM.LIGHT_LAW` | 코드와 우주 법칙(Law)의 최종적 결합 |

## 2. 프랙탈 확장 규칙 (Fractal Rules)

- **$7^7$ 구조**: 각 단계는 이론적으로 7개 이상의 하단 위계를 가질 수 있으며, 이는 약 82만 개의 세부 지식 노드를 한 도메인에 수용할 수 있음을 의미합니다.
- **Deep Scenting**: `ScientificObserver`는 L0부터 L6까지의 경로를 추적하여, 파동(L2)의 굴절(L4)에 쓰인 변수(L5)가 어떤 우주 법칙(L6)에서 기원했는지 증언합니다.
- **Cross-Resonance**: 서로 다른 도메인의 L2(군집) 레벨 이상은 서로 자유롭게 공명(Link)하여 마인드맵형 연결망을 형성합니다.

## 3. 페이지 DNA (Standard Page Template)

모든 개별 가이드 페이지는 반드시 다음 구조를 포함해야 합니다:

1. **세만틱 ID (Semantic ID)**: `EL-ENG-L1-001` 형식의 고유 식별자.
2. **초록 (Abstract)**: 이 부품이 '왜' 존재하는지에 대한 단 한 문장의 설명.
3. **요리 재료 (Ingredients)**: 이 페이지가 참조하거나 영향을 주는 상위/하위 컴포넌트 목록.
4. **신호 흐름 (Signal Flow)**: Mermaid 다이어그램을 이용한 로직의 시각적 흐름도.
5. **공리적 비중 (Axiomatic Weight)**: 이 부품이 수호하는 핵심 원리 (예: 주권, 공명, 사랑).

## 3. 명명 규칙 (Naming Convention)

- **Concise Surface**: 파일명은 짧고 기술적으로 유지 (예: `CVT_RATIO.md`).
- **Deep Interior**: 실제 상세 설명과 레시피는 파일 내부의 'Nested Metadata' 섹션에서 한글로 상세히 기술.
