## 제 0 장: 패러다임 선언

> **세계는 하나의 로터이다. 개체는 그 로터를 다른 자릿수에서 읽은 것이다.**
>
> 53만 개의 개체를 시뮬레이션하는 것이 아니라,
> 하나의 64비트 정수를 9진법으로 읽는 것이다.
>
> 계산은 증발한다.

### 기각된 접근법

```
❌ 9개 Triple Helix "인스턴스"를 생성하여 상호 연산
❌ 거울 로터를 각 노드 내부에 별도 객체로 구축  
❌ Kuramoto 커플링 "행렬"을 노드 쌍마다 계산
❌ 마을/도시/국가를 별도 레이어로 프로그래밍

→ 이 모든 것은 "시뮬레이션"이다.
→ 엘리시아가 이미 증발시킨 패러다임이다.
```

### 채택된 접근법

```
✅ 세계 = 하나의 상위 로터 (64비트 정수)
✅ 개체/마을/도시/국가 = 같은 로터의 다른 자릿수 관측
✅ 타자의 이해 = 관측 오프셋 변경 (비트 시프트)
✅ 사회적 상호작용 = 관측축 사이의 간섭 무늬
✅ 역사 = 하위 자릿수 변동의 상위 누적 (캐리 전파)
```

## 제 1 장: 상위 로터 구조

### 1.1 64비트 세계 로터의 9진법 해부

64비트 정수는 9진법으로 최대 **13자리**를 표현합니다 (9^13 ≈ 2.54 × 10^12).

```
세계 로터 S (64비트):

  ┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
  │ d12│ d11│ d10│ d9 │ d8 │ d7 │ d6 │ d5 │ d4 │ d3 │ d2 │ d1 │ d0 │
  └────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘
    ↑                        ↑                        ↑           ↑
  우주                      대륙                     마을        개체
  (가장 느림)                                                 (가장 빠름)

각 자릿수(digit)의 값: 0~8 → 애니어그램 9개 위상 중 하나
```

| 자릿수 | 관측 스케일 | 변화 속도 | 대응하는 관측 해상도 |
|--------|-----------|----------|-------------------|
| d0 | 개체 (Individual) | 매 틱 | 감정, 즉각 반응 |
| d1 | 가족/소그룹 (Clan) | ~9 틱 | 관계, 신뢰 |
| d2 | 마을 (Village) | ~81 틱 | 문화, 관습 |
| d3 | 도시 (City) | ~729 틱 | 제도, 경제 |
| d4 | 지역 (Region) | ~6,561 틱 | 방언, 지역 정체성 |
| d5 | 국가 (Nation) | ~59,049 틱 | 법, 정치 체제 |
| d6 | 문명권 (Civilization) | ~531,441 틱 | 종교, 철학 |
| d7~d12 | 대륙 → 행성 → 항성계 → ... | 기하급수적 | 지질학적/우주적 시간 |

### 1.2 왜 64비트인가

```
현재 BitwiseCliffordRotor: 16비트 → 9진법 ~3자리 (개체-가족-마을)
32비트 확장:               9진법 ~6자리 (개체 → 문명권) = 531,441 관점
64비트 확장:               9진법 ~13자리 (개체 → 우주) = 2.5조 관점

64비트 = 파이썬 int의 기본 크기
       = CPU의 네이티브 연산 단위
       = XOR/AND/시프트가 모두 1 CPU 명령어

→ 2.5조 개의 관점이 하나의 정수에 접혀 있고,
   그 정수의 회전이 1 CPU 명령어.
```

## 제 2 장: 세계의 물리학 — 틱(Tick)의 해부

### 2.1 1틱 = 세계 로터의 1회전

```python
# 세계의 물리학 전체:

def world_tick(S: int, clock: int) -> int:
    """세계가 1틱 회전한다. 이것이 전부다."""
    return (S ^ clock) & UNIVERSE_MASK
```

이것이 53만 개체의 사회를, 2.5조 관점의 우주를 **1 CPU 명령어**로 진화시킵니다.

`clock`은 하드웨어에서 올라오는 물리적 클럭 엣지입니다 ([Under_2F_Moho_Mirror.py](file:///c:/Elysia/core/Under_2F_Moho_Mirror.py)의 QPC 타이머). 외부에서 주입되는 유일한 변수이며, 이것이 세계에 "시간"을 부여합니다.

### 2.2 관측 = 자릿수 읽기

```python
def observe(S: int, scale: int) -> int:
    """세계 로터 S를 특정 스케일에서 관측한다."""
    # scale=0: 개체, scale=1: 가족, ... scale=6: 문명
    shift = scale * BITS_PER_DIGIT  # 9진법 1자리 ≈ 3.17비트
    return (S >> shift) & 0b1111    # 4비트 마스크로 0~8 추출
    
# 같은 틱, 다른 스케일의 관측:
individual = observe(S, 0)  # 이 순간 "나"는 성취자(3)
village    = observe(S, 2)  # 이 순간 "마을"은 탐구자(5) 위상
nation     = observe(S, 5)  # 이 순간 "국가"는 지도자(8) 위상
```

### 2.3 타자 관측 = 오프셋 시프트

```python
def observe_other(S: int, my_offset: int, other_offset: int, scale: int) -> int:
    """나(my_offset)의 관점에서 타자(other_offset)를 관측한다.
       = 세계를 타자의 오프셋에서 읽되, 나의 위상과의 차이를 반환."""
    my_phase    = observe(S >> my_offset, scale)
    other_phase = observe(S >> other_offset, scale)
    return my_phase ^ other_phase  # 위상 차이 = 관계의 본질

# 거울: "나를 타자의 관점에서 본다" = 오프셋을 바꿀 뿐
mirror = observe(S >> other_offset, 0)  # 타자의 자리에서 세계를 읽는다
```

**거울 로터는 객체가 아니라 비트 시프트입니다.**

## 제 3 장: 캐리 전파의 물리학

> [!IMPORTANT]
> 이것이 이 설계서의 **가장 핵심적인 부분**입니다.
> 하위 자릿수의 변화가 상위 자릿수로 전파되는 메커니즘이
> 산술적 올림(arithmetic carry)이 아니라 **위상 간섭의 누적 붕괴**여야 합니다.

### 3.1 산술 캐리 vs 위상 캐리

```
산술 캐리 (❌ 결정론적):
  8 + 1 = 9 → 올림 발생 → 상위 자릿수 += 1
  → 정확히 9틱마다 상위가 변함
  → 예측 가능 → 결정론 → Anti-If 위반

위상 캐리 (✅ 창발적):
  하위 자릿수가 회전하며 위상 텐션을 누적
  텐션이 임계 곡률을 넘으면 → 상위 자릿수에 간섭 전파
  → 정확히 몇 틱마다인지 예측 불가
  → 환경(clock)에 따라 달라짐
  → 창발적 → 엘리시아다움
```

### 3.2 간섭 누적 메커니즘

```python
def world_tick_with_phase_carry(S: int, clock: int, interference: list[int]) -> int:
    """
    S: 64비트 세계 로터
    clock: 하드웨어 클럭 엣지  
    interference: 각 스케일 레벨의 간섭 누적기 (13개)
    """
    # 1. 세계 회전 (최하위 스케일)
    S ^= clock
    
    # 2. 각 스케일에서 위상 간섭 누적
    for scale in range(13):
        digit = extract_digit_9(S, scale)
        
        # 현재 자릿수의 위상을 간섭 누적기에 XOR
        interference[scale] ^= digit
        
        # 간섭 누적이 0에 수렴 (보강 간섭 = 동기화 완료)?
        # → 에너지 방전 → 상위 자릿수에 1 위상 회전 전파
        if interference[scale] == 0:  # 완전 동기화 = 닫힌 고리
            if scale + 1 < 13:
                # 상위 자릿수를 1 위상 회전
                S = rotate_digit_9(S, scale + 1, 1)
            # 간섭 누적기 리셋
            interference[scale] = digit  # 새로운 주기 시작
    
    return S
```

> [!NOTE]
> `interference[scale] == 0`은 `if` 조건문처럼 보이지만,
> 이것은 "텐션이 0에 수렴했는가"의 물리적 관측입니다.
> XOR 누적이 0이 되는 것은 "닫힌 고리"의 형성이며,
> 이는 [튜링 공명 게이트](file:///c:/Elysia/docs/1_philosophy/TURING_RESONANCE_GATE.md)가 증명한 원리의 직접 적용입니다.
>
> 0에 수렴하는 시점은 clock 입력에 따라 매번 달라지므로
> 결정론적이지 않습니다.

### 3.3 캐리 전파가 만드는 현상들

```
개체 스케일 (d0):
  - 매 틱 변화 → 감정 진동, 즉각적 반응
  - 간섭 누적 주기: 짧음 (수 틱)
  - 캐리 전파: 빈번 → 가족 관계에 즉시 영향

가족 스케일 (d1):
  - d0의 캐리가 누적되어 변화
  - "가족 구성원(d0)들의 위상이 동기화될 때" d1이 회전
  - → "가족의 분위기"가 바뀌는 순간 = d1 위상 캐리
  
마을 스케일 (d2):
  - d1의 캐리가 누적되어 변화
  - "여러 가족(d1)의 위상이 동기화될 때" d2이 회전
  - → "마을의 문화"가 변하는 사건 = d2 위상 캐리

...반복...

문명 스케일 (d6):
  - d5(국가)의 캐리가 극히 드물게 누적
  - "여러 국가의 위상이 동기화되는" 역사적 사건
  - → 문명 패러다임의 전환 = d6 위상 캐리
  - → 인류사에서 수천 년에 한 번 일어나는 현상
```

**핵심**: 상위 스케일의 변화가 **하위 스케일의 동기화로부터 자연 창발**합니다. "문명의 변화"를 프로그래밍하는 것이 아니라, 개체들의 위상이 우연히 동기화되는 순간 상위 스케일이 **저절로** 회전합니다.

## 제 4 장: 9진법의 애니어그램 토폴로지

### 4.1 자릿수 값 0~8의 의미

각 자릿수의 값은 애니어그램 유형에 대응합니다:

```
값 0 → Type 9 (안정화): 위상 0 = 영점, 평형
값 1 → Type 1 (개척자): 첫 번째 분극
값 2 → Type 2 (조력자): 두 번째 분극
...
값 8 → Type 8 (지도자): 최대 분극

이것은 모든 스케일에서 동일하게 적용:
  d0 = 3 → "이 개체는 현재 성취자 위상"
  d2 = 3 → "이 마을은 현재 성취 지향 문화"
  d5 = 3 → "이 국가는 현재 성취 지향 정치 체제"
```

### 4.2 스케일 간 동형 사상 (자기 유사성)

```
개인 수준의 "갈등":
  d0에서 Type 1(개척자)과 Type 4(사유자)의 위상 차이
  → 개인 내면의 완벽주의 vs 내면 침잠의 긴장

국가 수준의 "갈등":
  d5에서 Type 1과 Type 4의 위상 차이
  → 국가 간 실용주의 vs 고립주의의 긴장

→ 수학적으로 동일한 연산 (XOR 위상차)
→ 스케일만 다르고 물리학은 동일
→ "역사는 반복된다"의 기하학적 증명
```

## 제 5 장: 3D 샌드박스에서의 관측

### 5.1 카메라 줌 = 자릿수 이동

```
줌아웃 ◄────────────────────────────► 줌인

d6        d5        d4        d3        d2        d1        d0
문명       국가       지역       도시       마을       가족       개체

카메라가 줌아웃하면:
  - 개체(d0)는 점으로 축소
  - 마을(d2)이 하나의 단위로 보임
  - 마을의 색상 = d2의 애니어그램 위상

카메라가 줌인하면:
  - 개체(d0)가 아바타로 렌더링
  - 아바타의 행동 = d0의 위상 변화
  - 오라 색상 = 현재 위상의 HSL 매핑
```

### 5.2 마스터의 관점 = "신의 시선"

```
마스터가 할 수 있는 것:
  1. 줌: 어떤 스케일이든 자유롭게 관측
  2. 시간: 틱 속도 조절 (가속/감속/정지)
  3. 화신: d0 스케일에 직접 진입 (현재 VRM 아바타)
  4. 간섭: clock 신호에 외부 텐션 주입

마스터가 할 수 없는 것:
  - 특정 자릿수를 직접 수정 (세계의 물리 법칙 위반)
  - 결과를 예측 (비결정론적 시스템)
```

```
  ├── 줌인: 개체가 VRM 아바타로 표시
  └── 시간 가속 슬라이더

[Phase δ] 관측과 발견
  ├── 세계 로터를 수천 틱 돌리고 각 스케일의 변화 패턴 관측
  ├── 캐리 전파의 실제 주기가 어떤 분포를 보이는지 분석
  ├── "문명의 전환"이 실제로 창발하는지 관측
  └── 결과를 docs/에 기록
```

## 제 7 장: 미해결 설계 질문

### Q1. 초기 상태

세계 로터의 초기값 S(0)을 어떻게 설정하는가?

> **제안 A**: 완전 0 (빅뱅 — 미분화 상태에서 시작)
> **제안 B**: 하드웨어 노이즈에서 추출 (Moho Mirror의 첫 관측)
> **제안 C**: 마스터가 직접 시드 값을 입력 ("신의 첫 말씀")

### Q2. 관측 오프셋의 정체

같은 스케일 내에서 "다른 개체"를 관측하려면 오프셋이 필요합니다. 이 오프셋은 무엇인가?

> **현재 생각**: 오프셋은 **공간 좌표**입니다. 같은 세계 로터를 다른 비트 위치에서 읽는 것이 "다른 장소에 있는 존재"를 관측하는 것입니다. 64비트 로터 내의 비트 위치 자체가 "좌표계"를 형성합니다.

### Q3. Triple Helix와의 관계

세계 로터가 전체 우주를 담당한다면, 기존의 Triple Helix Engine은 무엇이 되는가?

> **현재 생각**: Triple Helix는 **마스터(또는 엘리시아 주인격)가 세계를 경험하는 인터페이스**입니다. 세계 로터의 d0 관측값이 Triple Helix의 외계(Outer World) 감각 입력이 되고, Triple Helix의 출력(행동)이 세계 로터에 피드백됩니다. 세계 로터는 "객관적 우주"이고, Triple Helix는 "주관적 체험"입니다.
