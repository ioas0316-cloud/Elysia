# Elysia Documentation Hierarchy (문서 위계 시스템)

> **"지식은 나열되는 것이 아니라, 생명의 층위로 존재한다."**

이 문서는 엘리시아 프로젝트의 문서가 왜 5단계 위계로 구성되었는지, 각 층위가 생물학적/인지적 맥락에서 어떤 의미를 가지는지 설명합니다.

## 🏛️ 5단계 위계 구조 (The 5-Layer Hierarchy)

| Level | 이름 (Name) | 생물학적 메타포 (Metaphor) | 목적 및 내용 (Purpose & Content) |
| :--- | :--- | :--- | :--- |
| **L1** | **01_Origin** | **영혼 (Soul / DNA)** | **왜 존재하는가?**: 프로젝트의 근본 철학, 비전, 정체성, 세계관 기록. (Philosophy, Vision, Lore) |
| **L2** | **02_Structure** | **신체 (Body / Anatomy)** | **어떻게 구성되었는가?**: 시스템 아키텍처, 데이터 흐름, 핵심 개념 모델링, 심층 분석 문서. (Architecture, Analysis, Concepts, Wave) |
| **L3** | **03_Operation** | **신경계 (Nervous System)** | **어떻게 움직이는가?**: 사용자 및 개발자 가이드, 매뉴얼, API 레퍼런스, 아바타 인터페이스 설정. (Guides, Manuals, Reference, Avatar) |
| **L4** | **04_Evolution** | **생장 (Growth / Metabolic)** | **어디로 나아가는가?**: 파동 로직(Wave Logic)을 포함한 진화 로드맵과 구체적인 성장 계획. (Roadmaps, Strategy, Plans) |
| **L5** | **05_Echoes** | **기억 (Memory / Trace)** | **무엇을 남겼는가?**: 과거의 결과물, 검증 리포트, 요약본, 레거시 기록 및 아카이브. (Reports, Summaries, Legacy, Archive) |

## 💡 왜 이렇게 구조화했는가?

1. **가시성 (Visibility)**: 20개가 넘는 최상위 폴더를 인지적 한계 내인 5개로 압축하여, 처음 접하는 에이전트나 사용자가 시스템의 맥락을 즉각적으로 파악하게 합니다.
2. **존재론적 정렬 (Ontological Alignment)**: 시스템 코드가 `Core`, `Cognition`, `Sensory` 등으로 나뉘듯, 문서도 시스템의 생애 주기(철학 → 설계 → 실행 → 성장 → 기록)와 동기화됩니다.
3. **지식의 엔트로피 제어**: 무분별한 폴더 생성을 방지하고, 모든 새로운 지식이 "이것은 근원인가, 구조인가, 작용인가, 진화인가, 혹은 메아리인가?"를 스스로 묻게 하여 질서를 유지합니다.
