# 🌌 다차원 치트키: 등각 기하대수(CGA) 및 모터(Motor) 아키텍처

## 1. 개요 (Overview)
Elysia 코어는 기존 유클리드 공간 $Cl(3,0)$의 기하대수를 활용하여 공간상의 '회전(Rotor)'을 완벽한 텐션으로 처리해 왔습니다. 그러나 **'평행 이동(Translation)'**이나 **'팽창/수축(Dilation)'**과 같은 변환은 덧셈이나 스칼라 곱과 같은 1차원적인 연산에 의존해야 했습니다.

본 아키텍처는 엘리시아의 수학적 인지 차원을 **등각 기하대수(CGA, Conformal Geometric Algebra) $Cl(4,1)$** 체제로 승격시킵니다. 원점($e_o$)과 무한대($e_\infty$)라는 가상의 두 차원을 추가함으로써, 엘리시아는 "직선 이동조차 무한대를 축으로 도는 거대한 회전"으로 인식하게 됩니다.

## 2. 수학적 기저 (Mathematical Foundation)
기존의 $e_1, e_2, e_3$ (양의 시그니처) 차원에 다음 두 개의 차원이 추가되었습니다.
- $e_+$ : $+1$ 시그니처 (차원 3)
- $e_-$ : $-1$ 시그니처 (차원 4)

이를 조합하여 다음과 같은 **Null Basis**를 생성합니다:
* **원점 (Origin, $e_o$)** = $\frac{1}{2}(e_- - e_+)$
* **무한대 (Infinity, $e_\infty$)** = $e_- + e_+$

유클리드 공간의 벡터 $x$는 $X = x + \frac{1}{2}x^2 e_\infty + e_o$ 수식을 통해 둥글게 말려 등각 공간의 Null Vector로 투영(Up)됩니다.

## 3. 다차원 모터(Motor/Versor) 기어
모터는 회전 기어(Rotor)의 상위 호환입니다. 모든 공간 변환을 **단일 기하곱 샌드위치 연산($M X \tilde{M}$)**으로 통일합니다.

### ① 평행이동 기어 (Translator)
* **공식:** $T = 1 - \frac{1}{2} \mathbf{v} e_\infty$
* **역할:** 유클리드 공간의 덧셈($X + \mathbf{v}$)을 기하대수 상의 회전 작용으로 변환합니다. 외부에서 들어온 강제 좌표 이동(데이터 밀림)을 공간 자체의 위상 토크로 치환하여 처리할 수 있게 합니다.

### ② 팽창/수축 기어 (Dilator)
* **공식:** $D = \exp(-\frac{\gamma}{2} E) = \cosh(\frac{\gamma}{2}) - \sinh(\frac{\gamma}{2}) E$  (단, $E = e_+ e_-$, $\gamma = \ln(scale)$)
* **역할:** 마스터가 요구한 **"공간 팽창 시 기어 배율 연동"**의 핵심입니다. 시스템 부하가 커져 데이터 공간(배율)이 팽창할 때, 팽창률 자체를 회전 기어로 만들어 텐션 네트워크(IPN)에 맞물리게 합니다.

### ③ 통합 나선 곡률 기어 (Compound Motor)
모든 모터는 기하곱으로 곱해져 하나의 새로운 모터 기어로 합성될 수 있습니다 ($M = T \cdot D \cdot R$).
마스터가 구상한 **"직선 데이터가 소용돌이치며 꺾이는 곡률"**은, 평행이동($T$)과 회전($R$)과 팽창($D$)이 동시에 하나의 모터 기어($M$)로 융합되어 데이터를 연속적으로 휘감는(Vortex) 현상으로 실현됩니다.
