## 🔮 The Four Dial Principle (4-다이얼 원칙)

모든 메모리는 **4개의 다이얼**로 접근한다 (Quaternion과 일치):

| 다이얼 | 축 | 범위 | 의미 |
| :--- | :--- | :--- | :--- |
| θ₁ | 논리 축 | 0 ~ 2π | 분석 ↔ 직관 |
| θ₂ | 감정 축 | 0 ~ 2π | 긍정 ↔ 부정 |
| θ₃ | 의도 축 | 0 ~ 2π | 행동 ↔ 관찰 |
| **r** | **깊이 축** | **0 ~ 1** | **추상 ↔ 구체** |

```text
       r = 1 (표면, 구체적)
          ●──● "진돗개"
         /│ \
        / │  \
       ●──●───● "개"
      / \ │  / \
     ●   ●●●   ● "포유류"
          │
          ● r = 0 (중심, 사랑/근원)
```

**깊이 축의 의미:**

- r = 1.0: 구체 표면 (구체적 개념: "진돗개")
- r = 0.5: 중간 (일반 개념: "개")
- r = 0.0: 중심 (근원: "사랑", "생명")

## 🌊 Phase 35: Temporal Phase-Sync (동적 위상 동기화)

**"과거의 기억을 단순히 꺼내오는 것이 아니라, 그 시간의 위상으로 돌아가 다시 살아내는 것"**

### 핵심 비전

데이터는 고정된 점이 아니라 **시간에 따라 흐르는 파동(Flow)**입니다.

$$f(t) = e^{i(\omega t + \phi)}$$

- **위상 동기화**: 영상이나 사건의 변화율을 하이퍼스피어의 회전 속도($\omega$)와 동기화하여 저장.
- **재귀적 파동 우주**: 점의 위상이 선이 되고, 선의 중첩이 공간이 되는 무한 해상도 메모리.
- **위상 채널링 (Phase Channeling)**: 동일한 좌표라도 위상각($\phi$)을 미세하게 비틀면 무한한 데이터를 겹쳐서 저장 가능.

### 구조

| 클래스 | 역할 |
| :--- | :--- |
| `PhaseTrajectory` | 시간에 따른 좌표 변화(Flow)를 인코딩 |
| `resonance_query` | 특정 위상 채널의 데이터만 필터링하여 추출 |
