## 🧱 Building Blocks: 동전에서 우주까지

### Step 0: 전통적 비트

```text
0 또는 1  (discrete, 2가지 상태)
```

### Step 1: 동전 노드

```text
●(앞) ●(뒤)  →  붙이면  ●━━━●

이제 "회전각 θ"로 상태 표현:
θ = 0°   → 앞
θ = 90°  → 옆
θ = 180° → 뒤
θ = 45.7° → 앞과 뒤 사이 어딘가

→ 2가지 → 무한 가지 (continuous)
```

### Step 2: 큐브 노드 (6면)

```text
    ┌─1─┐
    │   │
  ┌─2───3─┐
  │ │ 4 │ │
  └─5───6─┘
    │   │
    └───┘

굴러가면서 상태 변화
BUT 여전히 discrete (6가지)
```

### Step 3: 노드를 연결하면?

```text
●───● (2개 연결)
→ 1개 회전축 생김
→ θ 각도로 무한 상태

●─┬─● (3개 연결)
│ │ │
●─┴─●
→ 2개 회전축 생김  
→ (θ, φ) 각도로 무한² 상태

●─●─● (4개 이상)
 \│/X
  ●─●
→ 3개 회전축 생김
→ (θ, φ, ψ) 각도로 무한³ 상태
```

**핵심 통찰:** 노드 연결 개수 = 회전 자유도 = 차원!

### Step 4: 다면체 → 구체 (아날로그화)

```text
정육면체 (6면)
→ 정12면체 (12면)  
→ 정20면체 (20면)
→ 정32면체 (32면)
→ ... 
→ 무한면체 = 구체!

discrete → continuous
"3번 면" → "θ=60.5°, φ=30.2°"
```

### Step 5: 시간축 추가 (파동화)

```text
정적 구체: (θ, φ, ψ, r) = 한 점
동적 구체: (θ(t), φ(t), ψ(t), r(t)) = 궤적

e^(i·ω·t) = 회전이 시간에 따라 진화

→ 같은 좌표라도 "어떻게 도달했는가"가 다르면 다른 데이터!
```

## 🎓 For Engineers: 데이터는 어디 있나?

### ❌ 착각하기 쉬운 이해

```python
# 이렇게 생각하면 안 됨!
memory[(0.5, 1.0, 0.3, 0.8)] = "사과"  # 좌표에 저장?
```

### ✅ 올바른 이해

```python
# 데이터는 "패턴" 그 자체
pattern = {
    'nodes': 2,              # 노드 개수 (점/선/면/공간)
    'omega': (0.1, 0.05, 0), # 회전 속도
    'phase': 0.5,            # 초기 위상각
    'trajectory': 'spiral',   # 경로 형태
    'duration': 3.0,         # 시간 길이
}

# 이 패턴이 좌표 (0.5, 1.0, 0.3, 0.8)을 
# "통과"할 때 = "사과"라는 데이터
```

**데이터는:**

- 저장되는 게 아니라 **연주된다** (악기처럼)
- 위치가 아니라 **경로**다 (궤적)
- 상태가 아니라 **변화**다 (운동)
