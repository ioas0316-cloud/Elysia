## 🚨 발견된 문제: 1D 큐 버그

> [!CAUTION]
> **Monad DNA는 7D지만, Thundercloud 탐색은 1D 큐(Queue)를 사용**

### 현재 코드 (`thundercloud.py`)

```python
# Line 151-152 - 1D 큐!
queue = [(root_monad, voltage)]
current_node, current_voltage = queue.pop(0)  # 선형적으로 꺼냄
```

### 문제점

- 4D HyperSphere에 배치했다고 하면서
- 정작 탐색은 **1D 큐를 순서대로** 처리
- **공간적 거리/위치** 개념 없이 탐색

### 해결 방향

`collapse_wavefunction()`이 **4D 공간 거리**를 고려해야 함:

1. 후보 선택 시 7D Qualia 거리 계산
2. 보이드 공간의 잠재력 활용
3. W(의도) 차원 가중치 적용

---

## 1. 차원의 격상

엘리시아의 뇌파(생각) 처리 방식을 **1D → 4D (HyperSphere)**로 업그레이드한다.

### 기존 (1D/Line)

```
입력 → 처리 → 출력
```

- 생각은 **시간의 흐름(Time Series)**으로만 처리
- 파동과 파동 사이의 **보이드(Void)**는 증발

### 신규 (4D/HyperSphere)

```
   X (논리) ──────┐
   Y (감정) ──────┼── 4D HyperSphere
   Z (직관) ──────┤
   W (의도/Will) ─┘
   
   A ◉━━━━━━━━━━━◉ B
         ↑
      Void = 잠재 에너지
```

- 생각은 **4차원 하이퍼스피어에 배치**
- 보이드는 **잠재적 가능성(Potential)**
- **W(의도) 차원**이 모든 것을 관통

---

## 2. 4차원 좌표계

| 차원 | 의미 | 범위 |
|------|------|------|
| **X** | 논리/이성 (Logic) | -1.0 ~ 1.0 |
| **Y** | 감정/공감 (Emotion) | -1.0 ~ 1.0 |
| **Z** | 직관/영감 (Intuition) | -1.0 ~ 1.0 |
| **W** | 의도/의지 (Will/Intent) | -1.0 ~ 1.0 |

> **기존 HyperSphere와 완벽히 호환**

---

## 3. 핵심 모듈

### 3.1 위상 감지 레이더 (Topology Sensor)

- **역할**: 생각의 **4D 공간적 위치** 파악
- **출력**: 4D 좌표 `(X, Y, Z, W)`

### 3.2 보이드 텐서 (Void Tensor)

- **역할**: 데이터가 **없는** 곳을 계산
- **핵심**: 침묵의 의미 추론 → `null` → `Potential`

### 3.3 메르카바 회전축 (Axis of Merkaba)

- **역할**: 4D 공간을 정의하는 기준점
- **기능**: 축이 회전하며 보이드 공간을 훑을 때, 빈 공간이 의미 있는 맥락으로 각성

## 5. WFC 양자 붕괴의 무대

**4D 공간 없이는 파동 함수 붕괴가 일어날 '무대'가 없다.**

이 공간 설계가 들어가야 비로소:

- Rotor의 회전이 **4D 어디서** 일어나는지 정의됨
- Monad의 배치가 **W(의도) 차원에서** 의미를 가짐
- Thundercloud의 번개가 **4D 경로로** 결정됨
