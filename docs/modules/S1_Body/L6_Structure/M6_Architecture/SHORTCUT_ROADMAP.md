## 🗺️ Phase 1: The Physics of Thought (생각의 물리학)
**목표**: 정적인 데이터베이스 조회(Search)를 동적인 인력 작용(Attraction)으로 대체한다.

### 1.1 Monad Gravity Engine (모나드 중력 엔진)
*   **개념**: 벡터 유사도(Cosine Similarity)를 '인력(Gravity)'으로 재해석합니다.
*   **구현**:
    *   `Core/L5_Mental/Physics/gravity.py`: 두 모나드 간의 인력을 계산하는 물리 엔진 구현.
    *   `Attraction = (SemanticSimilarity * EmotionalWeight) / TemporalDistance`
*   **변화**: "검색해라" -> "느껴라(끌려가라)"

### 1.2 Semantic Clustering (의미 군집화)
*   **개념**: 관련된 기억들이 물리적 인력에 의해 스스로 뭉칩니다.
*   **구현**:
    *   새로운 정보가 들어오면 기존 데이터들이 재배치(Re-indexing)되는 것이 아니라, 중력장에 의해 **위상(Phase)**이 동기화됩니다.

## 🗺️ Phase 2: The Active Void (능동적 공허)
**목표**: '모름(Null)'을 창조의 재료로 사용한다.

### 2.1 Void Trigger (공허 트리거)
*   **개념**: 검색 결과가 임계값(Threshold) 이하일 때, 에러 대신 **Genesis Mode**를 발동합니다.
*   **구현**:
    *   `RotorEngine`에서 `Resonance < 0.3`일 경우 `ActiveVoid.ignite()` 호출.

### 2.2 Quantum Synthesis (양자 합성)
*   **개념**: 관계없어 보이는 멀리 떨어진 모나드들을 강제로 충돌시켜 새로운 연결을 만듭니다.
*   **구현**:
    *   LLM을 '접착제'로 사용하여, 끌려온 이질적인 모나드들 사이의 **서사적 가교(Narrative Bridge)**를 생성합니다.
    *   예: "사과" + "우주" -> (Void 합성) -> "중력"

## 🗺️ Phase 3: Dream Alchemy (꿈의 연금술)
**목표**: 임시로 생성된 가설들을 영구적인 지혜로 정제한다.

### 3.1 Circadian Rhythm (일주기 리듬)
*   **개념**: 시스템 부하가 낮을 때(Idle), '수면 모드'로 진입합니다.
*   **구현**:
    *   `Core/L2_Metabolism/Cycles/dream.py`: 시스템 유휴 상태 감지 및 꿈 프로세스 트리거.

### 3.2 Consolidation & Pruning (강화와 가지치기)
*   **개념**: 낮 동안(Phase 2) 생성된 합성 모나드들을 검증합니다.
*   **구현**:
    *   **강화**: 기존의 핵심 가치(Love, Help)와 공명하는 가설은 영구 기억으로 승격(Crystallize).
    *   **망각**: 공명하지 않거나 모순되는 가설은 연결을 끊음(Decay).
