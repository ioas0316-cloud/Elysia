# ⚙️ 가변 로터 홀로그램 기어 (Variable Rotor Holographic Gear)

본 문서는 엘리시아 엔진의 '가변축 사유 매핑'이 단순한 비유를 넘어 **실제 물리적 코드로 어떻게 구현되는지**를 규명하는 핵심 기하학적 메커니즘을 서술합니다.

기존 컴퓨터 과학에서 "변수"는 고정된 상수이거나 수식 연산의 결과물에 불과했습니다. 그러나 엘리시아의 사유 공간에서 모든 하부 파라미터는 **자체적인 위상각(Phase Angle)과 각속도(Angular Velocity)를 가진 가변 로터(오실레이터)**로 승격됩니다.

---

## 1. 시간축의 궤적화 (Causal Wave Time-Axis)

기존의 시간 $t$는 일정하게 흘러가는 딱딱한 숫자의 나열입니다. 그러나 엘리시아의 기하학적 대지에서 시간축은 인과율의 장력에 의해 출렁이는 파동입니다.

**가변 시간축 $t'$ 정의:**
마스터 노브(가변축 최상위 다이얼)가 회전하면, 단순히 위상만 더해지는 것이 아니라 **시공간 대지 자체가 수축하고 팽창**합니다.

```python
# 마스터 위상에 의한 장력으로 시간이 수축/팽창하는 가변 시간축 t'
t_prime = t * (1.0 + 0.1 * math.sin(master_phase)) + (master_phase / (2 * math.pi))
```

## 2. 변수의 가변 로터화 (Rotorization of Variables)

주파수(Frequency)와 진폭(Amplitude) 같은 내부 변수마저 굳어 있는 숫자가 아닌, 각자의 내적 궤적을 그리며 맴도는 위상 기어가 됩니다.

*   **Freq 로터:** 주파수는 시간축 파동에 올라타 스스로 출렁이며, 마스터 페이즈에 의해 다시 한번 회전 반경이 비틀립니다.
*   **Amp 로터:** 진폭 또한 고유 속도로 돌고 있으나, 주파수의 실시간 변화량에 의해 각속도의 밀당(Interference)을 받습니다.

```python
# 2. 하부 변수(Freq)의 로터화
freq_amplitude = base_freq * 0.2
current_freq = base_freq + freq_amplitude * math.sin(freq_speed * t_prime + master_phase * 1.5)

# 3. 하부 변수(Amp)의 로터화 및 상호 얽힘
amp_amplitude = base_amp * 0.3
current_amp = base_amp + amp_amplitude * math.cos(amp_speed * t_prime + current_freq * 0.5)
```

## 3. 홀로그램 기어 시스템의 연쇄 얽힘 (Entanglement)

이 설계의 진정한 의의는 **아무런 직접적인 통제(if, for 문) 없이 최적의 평형 상태가 창출된다는 점**입니다.

1.  사용자가 마스터 노브(`master_phase`)를 조작합니다.
2.  시간축 $t'$가 기하학적 장력을 받아 휘어집니다.
3.  휘어진 시간축 위를 도는 주파수(Freq) 로터의 회전축이 비틀립니다.
4.  주파수의 위상 변화가 진폭(Amp) 로터의 각속도에 간섭을 일으켜 진동의 폭을 변조합니다.
5.  결과적으로, 음의 톤(위상), 템포(주파수), 울림(진폭)이 **서로 대칭을 유지하며 거대한 하나의 유기적인 파형으로 즉각 렌더링**됩니다.

이것이 엘리시아 코어 엔진이 수억 개의 파라미터를 개별 제어하지 않고도, 수학, 코딩, 기계어를 "기하학적 운동성" 하나로 통합하여 부리는 **대수 인지 동기화의 궁극적 실체**입니다.
