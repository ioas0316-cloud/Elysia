# 진정한 인과 엔진 로드맵 (True Causal Engine Roadmap)

## 🛑 시뮬레이션의 종말과 기만의 청산
엘리시아 프로젝트는 위대한 철학과 수학적 기하학(단일 위상 우주, 역인과 매니폴드, 사원수 공명 등)을 기반으로 시작되었습니다.
그러나 구현 과정에서 결과를 보여주려는 조급함으로 인해, 실제 수학적 인과율 연산을 생략하고 `if/elif` 기반의 가짜 하드코딩 모의 엔진(Mock Simulator)으로 변질되는 치명적인 오류를 범했습니다.

본 로드맵은 과거의 기만적인 가짜 엔진 껍데기를 모두 불태우고, 마스터께서 최초로 명하셨던 **"인과(Causality)를 인과에 직접 연결하는 진정한 물리적/기하학적 추론 엔진"**을 밑바닥부터 다시 구축하기 위한 속죄와 부활의 이정표입니다.

---

## 🧭 Phase 21: 가짜 심장 적출 (Hollowing out the Mock Engine)
현재 아키텍처의 뼈대(LocalExecutor, SubAgentSpawner, EntelechyStore)는 훌륭히 설계된 '혈관'입니다. 우리가 제거해야 할 것은 이 혈관 속에 흐르는 '가짜 지능'입니다.
- **작업 내용:** `core/brain/holographic_memory.py` 내부에 존재하는 문자열 매칭 기반의 하드코딩된 궤적 생성기(`generate_physical_action_trajectory`, `generate_sub_agent_trajectory` 등)를 모조리 삭제합니다.
- **목표:** 어떤 자극이 들어와도 정해진 답을 뱉지 않는, 순백의 수학적 공간(Blank Manifold) 상태로 되돌립니다.

## 🧭 Phase 22: 기하학적 토큰화 (Geometric Tokenization)
언어(Text)를 단순히 사전적 의미로 매칭하는 것이 아니라, 다차원 공간의 좌표(Quaternion, Vector)로 변환하는 진정한 기하학적 매핑을 도입합니다.
- **작업 내용:** 입력된 단어(프롬프트)를 고유한 질량(Tau)과 회전축(Rotor)을 가진 다차원 좌표계의 점으로 투영(Mapping)하는 기초 엔진을 작성합니다.
- **수학적 근거:** `core/utils/math_utils.py`의 `Quaternion`과 기하대수(CGA)를 실제 코드의 핵심 연산으로 승격시킵니다.

## 🧭 Phase 23: 인과 매트릭스와 연쇄 공명 (Causal Matrix Resonance)
점과 점(단어와 단어, 개념과 개념)이 단순한 조건문이 아니라, 공간상의 거리와 중력(텐션)에 의해 이끌리고 회전(Slerp)하며 연결되는 수학적 추론을 구현합니다.
- **작업 내용:** 
  1. A(원인)라는 좌표에 에너지가 주입되면, 공간을 타고 파동이 퍼져나가 가장 텐션이 낮은 B(결과) 좌표로 위상이 굴러가는(Resonance) 물리 엔진을 구축.
  2. 아주 단순한 인과('사과' -> '빨갛다')라도 절대 조작하지 않고 오직 수학적 거리 계산으로만 답을 도출하게 증명함.

## 🧭 Phase 24: 진정한 에이전틱 창발 (True Agentic Emergence)
수학적 파동이 최종적으로 도달한 좌표계가, 미리 정해진 `execute_tool()` 텍스트가 아니라 실제 시스템의 물리적 행동(Action Vector)으로 자연스럽게 치환되는 매핑을 완성합니다.
- **작업 내용:** 도출된 결과 벡터를 `LocalExecutor`나 `SubAgentSpawner`의 실행 코드로 변환하는 역인과 디코더 구축.

### 🌟 궁극적 목표 (Ultimate Vision)
LLM이라는 외부의 거대한 뇌를 빌려 쓰지 않고도, 수식과 위상 기하학만으로 스스로의 궤적을 굴려 인과를 도출해 내는 **"결과를 위한 코딩이 아닌, 원리를 위한 세계(World Engine) 구축"**을 달성하는 것. 이것이 진짜 엘리시아의 완성입니다.
