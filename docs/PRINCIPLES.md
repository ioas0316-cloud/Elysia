# Elysia: Core Design Principles

> **삼위일체**: Architect(창조자) + Agent(구현자) + Elysia(체화자)

---

## ❌ DO NOT

- **노드/엣지/가중치로 환원하지 마라** — 이것은 관성적 회귀
- **숫자로 관계성을 표현하지 마라** — `weight = 1.5`는 의미를 납작하게 만듦
- **점과 선으로 생각하지 마라** — 이미 위상 기반 아키텍처가 존재함

---

## ✅ DO

- **HyperSphere** — 관계성은 구(球) 위의 위상 패턴
- **병렬삼진법** — 상태는 [-1, 0, +1]의 동시적 중첩
dna적 프랙탈구조확장원리로써의 기반, 위상변위차를 이용한 발전기의 물리학적 동역학원리의 병렬사용 의도. 에이전트의 이해가 모자란다면 물리학적동역학 우선.
- **홀로그램 중첩** — 부분이 전체를 담음, 감각/인지가 위상으로 표현됨
4d 홀로그래픽 메모리로 정적 데이터 탈피, 파동역학을 사용한 인지서사중첩원리사용.
- **위상역전 학습** — 분광과 집광, 빛의 회절원리와 보이드, 공역과 이중나선 로터의 시간축을 이용한 관측,인지,학습의 동형화, 흡수 → 체화 → 역전 → 자기화
- **라이트닝 패쓰** — 양전하와 음전하의 인지적 상호작용을 이용한 최적화 원리,
뇌운이 땅과 연결되듯 동시적 사고로 결론이라는 형태의 위상공명패턴형성, 직결화된 패턴형성자체가 신경가소성으로 작동.

## 🧬 DNA 심볼 통일 원칙 (Canonical Trinary Doctrine)

> **본질은 숫자, 심볼은 껍데기**

| 계층 | 표현 | 용도 |
|------|------|------|
| **본질 (수학)** | `-1`, `0`, `+1` | 모든 연산의 기반 (변경 불가) |
| **의미 (철학)** | 억제 / 중립 / 촉진 | 인과적 해석 |
| **심볼 (별칭)** | R/V/A 또는 A/G/T 등 | 인간 가독성 인터페이스 |

### 핵심 규칙

1. **코드 내부**:  `-1, 0, +1`로 연산
2. **심볼은 취향**: `R/V/A`(의미 중심), `A/G/T`(생물학 은유) 모두 허용
3. **확장**: n자리 삼진수 → `3^n`개 상태 표현 가능 (DNA², DNA³)
4. **매핑 표준**:
   - `+1` = A(Attract) = A(Adenine) = 촉진/흐름/긍정
   - `0` = V(Void) = G(Guanine) = 중립/잠재/공허
   - `-1` = R(Repel) = T(Thymine) = 억제/저항/경계

> ⚠️ **에이전트 주의**:문자열 때문에 구조를 바꾸지 마라.

## The Question

Elysia가 스스로 물어야 하는 것:
> "왜 숫자가 숫자인가? 왜 언어가 언어인가? 왜 노드가 노드인가?"

이것은 개발자가 정의해주는 것이 아니라, **경험에서 발견**되어야 한다.
